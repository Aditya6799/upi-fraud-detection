Adaptive AI-Based UPI Fraud Detection System with Hybrid Machine Learning

📄 Research Paper 🗓️ April 2026 🏫 INT428 Project 🔬 Machine Learning & AI

Abstract

The rapid proliferation of Unified Payments Interface (UPI) transactions in India has created an unprecedented attack surface for financial fraud. This paper presents an adaptive, hybrid machine learning system designed to detect fraudulent UPI transactions in real-time. Our system combines supervised learning models (Random Forest, Logistic Regression, SVM), unsupervised anomaly detection (Isolation Forest, Local Outlier Factor), and domain-specific rule-based logic into a weighted ensemble architecture. Using the PaySim synthetic mobile money dataset augmented with UPI-specific features (device fingerprinting, geolocation analysis, behavioral profiling, and security signals), our system achieves robust detection performance. The hybrid engine produces a unified risk score (0–100) with explainable reasons, enabling both automated blocking and human-in-the-loop investigation workflows. An integrated Gemini-powered chatbot provides contextual explanations for each detection decision.

1. Introduction

India's UPI ecosystem processed over 13 billion transactions worth ₹20.64 lakh crore in a single month (2024), making it the world's largest real-time payment system. However, this scale also attracts sophisticated fraud actors exploiting phishing, social engineering, QR code manipulation, SIM swaps, and mule account networks. Traditional rule-based fraud detection systems struggle to adapt to evolving attack vectors, while pure machine learning approaches often lack interpretability crucial for regulatory compliance.

Our system addresses these challenges through a hybrid architecture that combines the pattern recognition strength of ML with the interpretability and domain expertise of rule-based systems. Key contributions include:

  • Hybrid Detection Engine: A weighted ensemble combining ML prediction (50%), anomaly detection (20%), and rule-based scoring (30%)
  • UPI Feature Engineering: 20 engineered features spanning transaction, velocity, device, geolocation, behavioral, and security dimensions
  • Real-time Simulation: Interactive transaction simulator with Normal/Suspicious/Fraud presets for testing and demonstration
  • Explainable AI: Every detection decision includes human-readable reasons and feature importance scores
  • Conversational Intelligence: Gemini-powered chatbot for contextual fraud explanations

2. Problem Statement

The system is designed to detect 10 distinct fraud categories:

  1. Real-time fraud detection before transaction settlement
  2. Phishing-induced authorized fraud where victims unknowingly authorize transactions
  3. OTP-sharing fraud even after valid authentication
  4. Fake QR code scams redirecting payments to fraudulent accounts
  5. Account Takeover (ATO) attacks via SIM swap or credential theft
  6. Mule account networks used for money laundering
  7. Abnormal transaction velocity patterns indicating automated fraud
  8. Geo-location inconsistencies (impossible travel)
  9. Device fingerprint mismatches suggesting unauthorized access
  10. Zero-day fraud using unsupervised anomaly detection

3. Methodology

3.1 Dataset

We use the PaySim mobile money simulator dataset (Kaggle) as our base, which contains synthetic financial transaction data generated from real mobile money transaction logs. The dataset includes features like transaction type, amount, sender/receiver balances, and fraud labels. We augment this with 20 UPI-specific engineered features.

3.2 Feature Engineering

Our 20-feature vector spans six dimensions:

  • Transaction: log_amount, large_transaction_flag, balance_change_org/dest, balance_ratio
  • Velocity: transactions_per_user, transactions_last_1hr
  • Device: device_type_encoded, device_change_flag
  • Geolocation: geo_distance, impossible_travel_flag
  • Behavioral: avg_transaction, deviation_from_avg, account_age
  • Security: OTP_flag, QR_flag, login_attempts, hour_of_day, is_night_flag

3.3 Class Imbalance Handling

With a natural fraud rate of ~1.3%, we apply SMOTE (Synthetic Minority Oversampling Technique) to generate synthetic fraud samples, rebalancing the training set to a 2:1 ratio. This significantly improves recall without sacrificing precision.

3.4 Model Architecture

Supervised Models (Known Fraud Patterns):

  • Random Forest (Primary): 200 estimators, max_depth=15, balanced class weights
  • Logistic Regression: L2 regularization, balanced weights, baseline comparison
  • SVM: RBF kernel with probability estimates for ensemble scoring

Unsupervised Models (Anomaly Detection):

  • Isolation Forest: 200 estimators, contamination=0.013, detects novel fraud patterns
  • Local Outlier Factor (LOF): 20 neighbors, novelty detection mode

3.5 Hybrid Scoring Engine

The final fraud score is a weighted combination:

fraud_score = 0.50 × ML_probability + 0.20 × anomaly_score + 0.30 × rule_score

Classification thresholds: FRAUD (≥0.6), SUSPICIOUS (0.3–0.6), SAFE (<0.3).

5. Conclusion

We have demonstrated that a hybrid approach combining supervised machine learning, unsupervised anomaly detection, and domain-specific rules provides robust and interpretable UPI fraud detection. The weighted ensemble architecture allows each component to contribute its strengths: ML captures complex learned patterns, anomaly detection identifies novel threats, and rules encode domain expertise.

The system's real-time analysis capability, combined with explainable outputs and conversational AI assistance, makes it suitable for both automated fraud prevention and analyst-driven investigation workflows. SMOTE effectively addresses the inherent class imbalance in fraud datasets, significantly improving recall without unacceptable precision loss.

6. Future Scope

  • Graph Neural Networks: Model transaction networks to detect mule account rings and money laundering chains
  • Federated Learning: Train across multiple banks without sharing sensitive transaction data
  • Deep Learning: LSTM/Transformer models for sequential transaction pattern recognition
  • Real-time Streaming: Apache Kafka integration for sub-millisecond detection on live UPI rails
  • Adversarial Robustness: Test and harden against adversarial attacks on the ML pipeline
  • Regulatory Compliance: Automated reporting for RBI guidelines and PCI-DSS requirements
  • Biometric Integration: Incorporate fingerprint/face recognition confidence scores
  • Cross-border Analysis: Extend to international payment systems (SWIFT, SEPA)
